## Условия работы и процесс согласования

Разработка ведётся на хостинге исполнителя. На протяжении всей работы сайт доступен по рабочей ссылке — заказчик видит актуальное состояние в любой момент без задержек.

**Порядок согласования каждой итерации:** Исполнитель показывает результат → заказчик принимает или возвращает на доработку → после согласования итерация фиксируется и следующая начинается только после этого.

**Хостинг и домен — на стороне заказчика.** По итогам приёмки всех работ заказчик предоставляет собственный хостинг и домен. Исполнитель переносит готовый проект, настраивает и передаёт в полное распоряжение заказчика.   
**Важно уточнить:** хостинг должен поддерживать запуск Node.js — это необходимо для работы бэкенда. Если хостинг этого не поддерживает, бэкенд разворачивается на отдельном сервере.

2. Эндпоинт /apiКак работает интеграция с Bazon

Bazon — платформа поставщика автозапчастей. Она предоставляет два CSV-файла, которые автоматически обновляются.

### Файл товаров — partssever-site-products.csv

Содержит полный каталог запчастей. Основные поля:

| Поле в CSV | Что это |
| :---- | :---- |
| Наименование | Название детали |
| Категория | Раздел каталога (Кузов, Двигатель, Электрика и т.д.) |
| Группа | Подкатегория внутри раздела |
| Марка / Модель / Год / Модификация | Применяемость (например: BMW X5 2021 G05) |
| Двигатель | Тип двигателя (XB1, B58M, BEV и т.д.) |
| Положение / Сторона / Ориентация | Расположение детали: перед/зад, лево/право, верх/низ |
| Цвет | Код цвета BMW (например: C27 arktis-grau, C1X Sunset orange) |
| Артикул | Внутренний номер детали |
| Все артикулы | Все OEM-номера через разделитель |
| Основной артикул | Главный OEM-номер для поиска |
| Производитель | BMW, MINI и т.д. |
| Описание | Описание, маркировки, особенности |
| Изображения | Ссылки на фотографии (до 12 штук) |
| Статус | new / used / contract — состояние детали |
| Цена / Валюта | Цена в рублях |

### Файл доноров — partssever-site-carsrc.csv

Содержит список автомобилей-доноров — реальных машин, с которых сняты запчасти. Каждая деталь физически привязана к конкретному автомобилю.

| Поле в CSV | Что это |
| :---- | :---- |
| Модель авто / Кузов / Год | Идентификация донора (например: BMW X5 G05 2021\) |
| Двигатель | Тип двигателя донора |
| Руль | Левый или правый руль |
| КПП / Привод | Коробка передач и тип привода |
| VIN | Уникальный номер автомобиля |
| Цвет | Код цвета кузова донора |
| Статус | Активен / Продан |

**Кодировка:** при скачивании CSV файлы декодируются через iconv-lite — это предотвращает проблему с кириллицей при неправильной обработке кодировки.

**Резервирование:** при недоступности Bazon во время синхронизации сайт продолжает работать на последней успешной версии данных.

Ключевая техническая часть. Всё, что видит покупатель, берётся из двух CSV-файлов Bazon — ничего не вводится вручную.

### Итерация 1.4 — Карточка товара (Дни 6–9)

Страница каждой отдельной детали. Все данные — напрямую из CSV Bazon. Задача — дать покупателю исчерпывающую информацию и закрыть все вопросы до звонка.

**Что отображается:**

| Блок | Источник данных |
| :---- | :---- |
| Название детали | Наименование |
| Артикул и OEM-номер | Артикул \+ Основной артикул |
| Кросс-номера и маркировки | Все артикулы |
| Применяемость (модель, год, кузов, двигатель) | Марка \+ Модель \+ Год \+ Модификация \+ Двигатель |

| Состояние | Статус: Новая / Б/У / Контрактная |
| Цена | Цена \+ Валюта |
| Фотогалерея | Изображения (до 12 фотографий) |

**Блок «Информация о доноре»** — конкурентное преимущество сайта:

Покупатель видит, с какой конкретной машины снята деталь, — это повышает доверие и снижает количество вопросов.

**1\. Хостинг** Хостинг заказчика должен поддерживать Node.js. Если нет — бэкенд размещается на отдельном сервере, это нужно согласовать заранее.

**2\. Форма заявки на Этапе 1** На Этапе 1 кнопка «В корзину» есть, но полноценное оформление заказа появится только на Этапе 2\. Нужна ли уже сейчас минимальная форма «Оставить заявку» с полем телефона прямо на карточке товара — чтобы покупатели могли обращаться сразу? 

**4\. Параллельные каналы продаж** Если ведётся продажа через Авито, Дром или другие площадки параллельно с сайтом. Как синхронизируется наличие?

**5\. Контакты и соцсети** Какие соцсети подключаем в футер — нужны активные ссылки. Телефон и email для отображения на сайте.

### Вопрос, требующий отдельного решения

**Что происходит при появлении новой категории в Bazon**

На Этапе 1 категории в меню сайта зафиксированы вручную. Если Bazon добавит новую категорию — товары из неё загрузятся в базу, но в меню не появятся и покупатель их не найдёт.

Есть два варианта:

**Вариант А — Ручное управление (текущий подход)** Категории прописаны в коде. При появлении новой категории в Bazon исполнитель добавляет её в меню вручную. Полный контроль над дизайном, порядком и названиями. Требует обращения к исполнителю при каждом изменении. — работает в рамках 2 этапа

**Вариант Б — Полностью автоматически**… нужно продумать  
и включить отдельным направлением в работе.
